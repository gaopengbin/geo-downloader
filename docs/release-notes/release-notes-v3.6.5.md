# v3.6.5 - 历史性能、日志清理与地图量测

本次更新整合了两个测试版本的验证结果，重点改善大量下载任务下的历史记录性能、任务日志占用与清理可靠性，并新增地图距离和面积量测工具。

## 历史记录性能

- 下载历史由单个 JSON 文件迁移到 SQLite，新增、删除和更新记录不再反复读写全部历史。
- 历史记录改为分页加载，每页 50 条，大量记录下仍可快速打开和翻页。
- 数据库操作移至后台线程，任务完成时不再阻塞下载运行时。
- 任务日志写入与任务列表锁分离，多个并行任务不会因日志磁盘写入互相等待。
- 首次升级会自动导入旧版 `history.json`，成功后保留为 `history.json.migrated`。

## 日志生命周期与清理

- 单个任务日志最大保留 5 MiB，达到上限后会明确标记截断。
- 完成超过 7 天的日志会分批压缩为 gzip，历史页面仍可直接查看。
- 超过 90 天的日志会自动清理，下载历史记录继续保留。
- 删除单条历史时，先删除关联的 `.log` 和 `.log.gz`，成功后再删除记录。
- 清空历史时同步删除关联日志，并清理旧版本遗留的孤立任务日志。
- 日志删除失败时保留历史记录并显示错误；正在运行的任务日志不会被误删。
- 所有日志清理均不会删除已经下载的影像、瓦片或其他成果文件。

## 地图量测

- Leaflet 地图新增距离、面积和清除量测工具。
- 距离量测显示各分段长度及总长度。
- 面积量测显示面积及周长。
- 量测结果会保留在地图上，直到主动清除；再次点击当前工具或按 `Esc` 可取消绘制。
- 量测图层与下载选区完全独立，不会修改或清除已有下载范围。
- 蓝色虚线和高对比度结果标签可在明暗及不同类型底图上保持清晰。

## 升级说明

- 稳定版用户可通过应用内更新检查升级。
- 首次打开历史记录时如需迁移大量旧数据，可能会有一次性的短暂等待。
- 如需协助排查历史或日志问题，请保留 `%LOCALAPPDATA%/geo-downloader/history.db`、`history.json.migrated` 和相关任务日志。

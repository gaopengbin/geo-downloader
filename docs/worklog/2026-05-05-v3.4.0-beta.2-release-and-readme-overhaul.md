## 踩到的坑

### 1. Cloudflare Pages 不是 GitHub 自动集成

- 项目 `geodownloader` **手动 wrangler 部署**，git push 不会触发更新
- 必须 `wrangler pages deploy site --project-name=geodownloader --branch=main`
- **不要重复 `npm install -g wrangler`**：用 `npx --yes wrangler@latest <cmd>` 即可，本地缓存复用，免去全局安装的污染与权限问题
- `npx --yes` 跳过交互确认；不带 `--yes` 时 `npx wrangler@latest` 会卡在 "Need to install ... ok? (y)" 等待 stdin
- 部署成功后会同时给出 deployment-id 域名（如 `https://973609d1.geodownloader.pages.dev`）和正式域名 `geodownloader.pages.dev`
- token：本机已 `wrangler login`（OAuth），`npx wrangler whoami` 可查；scopes 必含 `pages:write`

### 2. Tauri Release workflow 缺 Node

- `tauri.conf.json` 的 `beforeBuildCommand: npm run build --prefix frontend` 需要 Node + frontend 依赖
- 默认 release.yml 只 setup Rust，导致 `tsc not found` / exit 127
- 修复：在 Rust cache 之后插入：
  ```yaml
  - uses: actions/setup-node@v4
    with:
      node-version: 20
      cache: npm
      cache-dependency-path: frontend/package-lock.json
  - working-directory: frontend
    run: npm ci
  ```

### 3. npm 10.8.2 macOS 偶发 "Exit handler never called!"

- `npm ci` 静默退出，node_modules 没生成，后续 `tsc/vite` 找不到 → exit 127
- 不是配置问题，是 npm 自身 bug
- 修复：用 bash 脚本重试 3 次，每次校验 `node_modules/.bin/vite` 存在
  ```bash
  for i in 1 2 3; do
    npm ci && [ -x node_modules/.bin/vite ] && exit 0
    rm -rf node_modules; sleep 5
  done
  exit 1
  ```

### 4. GitHub sub-issues API 422 错误

- `gh api -f sub_issue_id=$id` 422 "is not of type integer"
- 原因：`-f` 总是字符串，必须用 `-F`（typed）才会发整数
- 正确：
  ```powershell
  gh api -X POST /repos/$owner/$repo/issues/$parent/sub_issues -F sub_issue_id=$childId
  ```

### 5. GraphQL pinDiscussion 不存在

- 想程序化置顶 Discussion，但 GraphQL 没有 `pinDiscussion` mutation
- 必须在 web UI 手动 Pin

### 6. 二维码图片名不一致

- README 里手写 `wxq.jpg`，但 `assets` release 实际是 `wxq_sq.png`
- 验证方法：`gh release view assets --json assets --jq '.assets[].name'`
- 现有代码 (`frontend/src/lib/qr-assets.ts`、`site/index.html`、`static/index.html`) 全部用 `wxq_sq.png`，README 是孤儿
- 教训：写 README 引用 release assets 时一律先 `gh release view` 校对文件名

### 7. PowerShell git stash 残留

- `git stash push -m "x" .github/copilot-instructions.md` 报 "No valid patches in input" 但仍然 "Saved working directory and index state"
- pop 时把 working tree 改动应用回来，多了不该有的 `src-tauri/Cargo.toml` CRLF 改动
- 教训：跨分支搬运单文件，优先用 `git checkout <src-branch> -- <file>`，不要走 stash

### 8. PowerShell 处理 git/wrangler stderr

- git 在 stderr 输出 LF/CRLF warning 时，被 PowerShell 当 NativeCommandError 抛出
- `2>&1 | Out-String` 可以收住，但有时 `Select-String` 串联后又被吞
- 教训：长命令分步走，关键 diff 用 `git diff --stat` 而不是 `git diff` 避免被 warning 截断

### 9. CI workflow 里 `gh run rerun --failed` 时机

- 当前 run 还在 `in_progress` 时不能 rerun（"This workflow is already running"）
- 必须等当前 run 整体 completed（即使其中部分 job 是 failed）才能触发
- release job `needs: build`，4 档全成功才会跑

### 10. PowerShell 单字符变量名 + 复杂 jq 过滤参数

- `gh run view ... --jq '.jobs[] | select(.name | contains("macOS-x64")) | .databaseId'` 在 PowerShell 反引号里嵌入失败
- 退路：把整个 JSON 拉回来 `ConvertFrom-Json` + `Where-Object` 过滤
  ```powershell
  $jobs = gh run view ... --json jobs | ConvertFrom-Json
  $target = $jobs.jobs | Where-Object { $_.name -like '*macOS-x64*' }
  ```

### 11. CI annotations 里的 Node 20 deprecation warning 不是项目问题

- GitHub 平台层面公告：`actions/checkout@v4` / `actions/setup-node@v4` 等 action 的 **运行时**（runner 跑 action 用的 Node）将在 2026-09-16 移除 Node 20
- 跟我们用 Node 20 编译前端无关（那个是我们自己 setup 的、跑在 runner 上的独立 Node）
- 等这些 action 升 v5 再 bump 即可，目前忽略

### 12. PowerShell `ConvertTo-Json` 中文转义已知坑（之前已记录，复用）

- 创建 GitHub Discussion / Release notes 时不要用 `ConvertTo-Json` 转中文 body
- 改用 `--notes-file file.md`（文件用 UTF-8 写入）或 GraphQL 直接 here-string

# v3.6.5-beta.1 - 历史记录性能测试版

这是一个预发布测试版本，重点验证大量批量任务长期运行时的历史记录和日志性能。

## 性能改进

- 下载历史由单个 JSON 文件迁移到 SQLite，新增、删除和更新记录不再反复读取并重写全部历史。
- 历史记录改为分页加载，每页 50 条，避免数千条记录同时渲染造成界面卡顿。
- 历史记录数据库操作移到后台线程，任务完成时不再阻塞下载运行时。
- 任务日志写入与任务列表锁分离，多个并行任务不会因为日志磁盘写入互相等待。

## 日志维护

- 单个任务日志最大保留 5 MiB，超过后会明确标记为已截断。
- 完成超过 7 天的日志会逐批压缩为 gzip，历史页面可直接查看压缩日志。
- 超过 90 天的日志文件会自动清理，但下载历史记录仍会保留。
- 删除历史记录时同步清理对应日志，同时保护仍在运行的任务日志。

## 数据迁移

- 首次打开历史记录时，旧版 `history.json` 会自动完整导入 SQLite。
- 导入成功后，原文件保留为 `history.json.migrated`，不会直接删除。
- 如果旧历史文件损坏，程序会保留原文件并报告错误，不会用空数据覆盖。

## 测试重点

- 使用上千条历史记录时，批量下载速度是否保持稳定。
- 首次升级后历史记录总数、顺序和翻页是否正确。
- 普通日志和压缩日志是否均可正常查看。
- 删除单条历史、清空历史及并行任务日志是否行为正常。

测试中如遇异常，请保留 `%LOCALAPPDATA%/geo-downloader/history.db`、`history.json.migrated` 和相关日志以便定位。

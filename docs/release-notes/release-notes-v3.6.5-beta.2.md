# v3.6.5-beta.2 - 历史日志清理修复

这是一个预发布测试版本，用于验证历史记录与任务日志的同步清理行为。

## 修复内容

- 删除单条下载历史时，会先删除关联的 `.log` 和 `.log.gz` 日志，成功后再删除历史记录。
- 日志删除失败时会保留历史记录并显示错误，避免界面提示成功但日志仍占用磁盘。
- 清空下载历史时，会同步删除全部关联日志，并清理旧版本遗留的孤立任务日志。
- 正在运行的任务日志会受到保护，不会被历史清理误删。
- 删除确认弹窗明确说明将同时删除关联日志，但不会删除已经下载的影像文件。

## 测试建议

- 删除一条包含日志的历史记录，确认记录及对应日志文件均被删除。
- 清空历史记录，确认旧的 `task_*.log` 和 `task_*.log.gz` 文件被清理。
- 在任务运行期间执行清理，确认活动任务能够继续运行且日志保持完整。
- 确认已下载的 GeoTIFF、瓦片和其他成果文件不受影响。
